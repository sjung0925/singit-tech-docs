# Singit 팔로잉 추천 로직

## 문제 정의
- 영상 플랫폼에서 사용자 활동 기반 팔로잉 추천 기능 필요
- 초기 서비스 단계에서 AI 모델 도입의 복잡성과 데이터 부족 문제
- 사용자 참여를 유도하기 위한 간단하고 효과적인 추천 시스템 요구

## 접근 방식
- AI 모델 대신 rule-based 방식 채택
- 사용자 활동 데이터를 기반으로 한 점수화 로직
- Redis 캐시를 활용한 최근 조회 프로필 제외 기능

## 추천 로직
- **사용 데이터**
  - 가창 횟수 (sing_count)
  - 재생 횟수 (play_count)
  - 청취 합계 (listen_sum)
  - 좋아요 합계 (like_sum)

- **점수 계산**
  ```sql
  ORDER BY sing_count DESC, play_count DESC, listen_sum DESC, like_sum DESC
  LIMIT 3
  ```

- **제외 조건**
  - 이미 팔로잉한 사용자
  - 추천 거부 설정된 사용자 (si_user_no_recommend)
  - 최근 조회한 프로필 (Redis 캐시 기반, 자정 만료)

- **캐시 관리**
  - Redis 키: `profile_views:{user_key}`
  - 만료: 다음 자정까지
  - 중복 조회 방지 및 추천 다양성 확보

## 설계 이유
- **Rule-based 선택 이유**
  - 초기 서비스에서 AI 모델의 데이터 요구량과 복잡성 부담
  - 유지보수 용이성: 로직 변경이 코드 수정으로 즉시 반영
  - 예측 가능성: 규칙 기반으로 결과 해석 및 디버깅 쉬움

- **현실적 판단**
  - 데이터 부족: 초기 사용자 활동 데이터 제한적
  - 리소스 효율: 간단한 SQL 쿼리로 충분한 성능
  - 빠른 구현: 개발 속도 우선

- **확장 가능한 구조**
  - 점수 가중치 조정 가능 (예: sing_count * 2 + like_sum)
  - 추가 데이터 통합 쉬움 (예: 팔로워 수, 최근 활동 시간)
  - 캐시 전략 확장 가능 (예: ML 기반 필터링 추가)

## 결과
- 간단한 rule-based 로직으로 효과적인 팔로잉 추천 구현
- 사용자 피드백 기반 점수 가중치 튜닝으로 개선 가능
- 초기 서비스에 적합한 유지보수성 높은 시스템 구축

### 관련 메소드
- `get_recommand_singmate`: 추천 싱메이트 조회
- `set_no_more_recommand_singmate`: 추천 거부 설정
- `save_profile_view_to_redis`: 프로필 조회 기록 저장
